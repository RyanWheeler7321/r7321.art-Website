The *r7321.art* website is finally live.

The goal from the start was to have a simple public home for updates, behind-the-scenes posts, finished work, and anything useful enough to share. The posts are allowed to be a little messy and wide-ranging because that matches the way the work actually happens week to week.

## Sidereal

{% cta "https://store.steampowered.com/app/2702590/Sidereal/", "Sidereal on Steam", "Store page" %}

{% image "/images/updates/website-launch/sidereal-01.webp", "Sidereal environment screenshot" %}

When I first set out to make a game with one hundred worlds, the concept sounded strong immediately. The actual reality was a lot messier, but still interesting.

{% image "/images/updates/website-launch/sidereal-02.webp", "Sidereal world screenshot" %}

This week was mostly patching systems together, playtesting, bug-fixing, and cutting content I originally wanted to add but no longer felt was worth the extra time.

{% image "/images/updates/website-launch/sidereal-03.webp", "Sidereal progression screenshot" %}

Building a Unity game with a huge number of scenes, unique assets, and multiple platform targets is a pain. Unity 6 and build profiles looked promising though, and I was already looking forward to using that workflow on whatever came next.

{% image "/images/updates/website-launch/sidereal-04.webp", "Sidereal atmospheric screenshot" %}

I never minded the Unity splash screen, but enough players seem to care that I will probably turn it off for future projects.

{% image "/images/updates/website-launch/sidereal-05.webp", "Sidereal item and environment screenshot" %}

{% callout "note", "AI note" %}
I experimented with current generative AI tools to speed up a few low-value asset tasks for *Sidereal*. It helped on filler work, but anything that actually needed a creative solution still had to be done by hand. The time savings are real in narrow cases. The creative value usually is not.
{% endcallout %}

The website turned out well enough to ship. It was all hand-built HTML, CSS, and JavaScript. I was already aware the particles might be slightly distracting and the mobile header alignment was not really solved, but it was good enough to get online and start using.

## Strego

{% image "/images/updates/website-launch/strego-01.webp", "Strego render" %}

{% cta "https://www.artstation.com/artwork/BkwP3D", "Strego on ArtStation", "Artwork page" %}

This was a speed sculpt after refamiliarizing myself with ZBrush. It was rendered in Blender EEVEE, procedurally textured in Blender, and finished with some compositing in Krita.

{% image "/images/updates/website-launch/strego-02.webp", "Second Strego render" %}

Strego is a strange character idea that has been sitting around for a few years. It pulls from Soulsborne games and horror-monster design, and I already had an ambitious future use in mind for it.
